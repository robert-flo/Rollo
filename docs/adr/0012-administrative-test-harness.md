# Build a separate administrative test harness

Administrative tasks will be validated through a dedicated administrative harness rather than extending the CLI/package Docker tester until it becomes ambiguous. Validation progresses through three distinct evidence levels: the fixture harness uses a temporary HOME for fast lifecycle checks; the Docker adapter provides reproducible container isolation; and the host runner is separately authorized and uses backups before reconciliation. Evidence from one level must never be presented as evidence from another. VM execution remains reserved for a future adapter. The first reference implementation is the SSH configuration task.
